Top Bar · Navbar

The top strip shows branding and quick actions.

  • Weather Pal logo — app icon + title. When an amendment is active the navbar tints a soft gold and pulsing ⚠️ AMENDMENT badges flank the time display.
  • Zulu / Local time — kept current with a 30-second tick. Matches UTC and the system clock of the display.
  • Published Xm ago — heartbeat indicator confirming the publishing server is still pushing fresh data. Publisher runs every 5 min, so thresholds are: green <10 min, yellow 10–20 min, red (pulsing) >20 min or no data yet. If this goes red, the server probably went idle — check the publishing automation.
  • ⚡ Detector — separate health pip for the standalone lightning service (which runs every minute, independent of the main 5-min publisher). Thresholds: green <2 min, yellow 2–5 min, red (pulsing) >5 min. Only appears after the first successful lightning.json fetch — hidden on initial page load. Hover for last-fetch age + flash count within 50 NM.
  • 🧪 Demo — 20 preset scenarios that replace live data with synthetic weather for testing layout and training responses. Active demo shows a yellow banner + yellow viewport border.
  • ❔ Help — opens this document.
  • 📡 Source — opens the army.mil weather page in a new tab.

Data Flow

Three independent legs, each with their own cadence:

  1. Publishing server → GitHub — every 5 min for the main publisher (data.json), every 1 min for the standalone lightning service (lightning.json).
  2. TV browser → GitHub — every 5 min for data.json, every 1 min for lightning.json. Re-renders in place (no page reload).
  3. TV browser → GitHub (code) — every 5 min. ETag check on index.html. Reloads the page only when a new dashboard version has been deployed.

Worst-case propagation from a server publish to the TV: ~5 min for main data, ~2 min for lightning. Worst-case from a code push to the TV: ~7 min.

Period Selector · AM / PM / N1

Three buttons below the navbar pick which forecast to display. The active period is auto-selected by local time and re-checks every minute:

  • 04:00 – 10:49 local → AM forecast
  • 10:50 – 17:14 local → PM forecast
  • 17:15 – 03:59 local → N1 (Night) forecast

You can tap any period to override; the selection sticks until the next auto-snap.

Solar / Lunar Strip

Single centered strip between the period selectors. Shows Sunrise, Sunset, EENT (End Evening Nautical Twilight — civil dark), Moonrise, Moonset, and % illumination. All times are local. Empty when the MEF for the active period doesn't include solar/lunar data.

METAR Strip

Stacked layout centered on the page:

  1. Raw METAR text — the full observation as delivered by aviationweather.gov. Colored by FAA flight category (VFR MVFR IFR LIFR).
  2. Decoded cells — Wind, Visibility, Ceiling, Temp, Dew Point, Altimeter.

Inline OLD DATA badge next to the raw text when the observation is more than 90 minutes old.

When the live METAR is stale or missing (e.g. aviationweather.gov is having issues for this station), the dashboard automatically falls back to synthesized conditions from the current GOLDFISH area-forecast segment and shows a cyan USING FORECAST · METAR IS STALE badge in place of OLD DATA. GOLDFISH is the correct fallback source because Cairns Army Airfield sits inside the GOLDFISH training area — its forecast already describes conditions over Cairns. In fallback mode the Cairns training panel and IFR training strip both read against the forecast rather than going UNKNOWN — the raw text line is prefixed FCST so it's obviously not an observation. The dashboard auto-switches back to the live METAR as soon as a fresh one is published.

Flight Level Winds

Horizontal strip showing wind direction/speed and temp for every forecast altitude inline (1k, 3k, 6k, 9k, etc.).

  • Wind text turns amber when speed ≥ 20 kt
  • Temp text turns blue when temp ≤ 4°C
  • Blue POSS ICE badge appended when temp ≤ 4°C (liquid water + cold = icing risk)

Lightning Banner

Sits above the Active Advisories card whenever the publisher's GOES-19 lightning fetcher detects flashes within 50 NM of KOZR in the past 10 minutes. Three tiers based on the closest strike:

  • ⚡ Within 5 NM — CRITICAL · pulsing red border. Surfaces Cairns's 5-NM operational rule. Banner reads "Operations Possibly Affected, Check Official Weather" — defers the authoritative halt call to the duty forecaster.
  • ⚡ Within 10 NM — VICINITY · static amber glow. Watch carefully; conditions are deteriorating.
  • ⚡ Within 25–50 NM only — DISTANT · subtle yellow strip. Storm cells are around but not yet in the operating area.

Collapsible — default collapsed to a thin one-line strip showing only the icon + tier label + chevron so it doesn't dominate the top of the dashboard during long stretches of distant activity. Click to expand for the full detail: closest strike line + per-ring count chips + EXPERIMENTAL badge. Preference persists across the nightly 03:00 reload via localStorage.

When expanded, the detail line shows the closest strike in the last 10 min — not the most recent — so a 49 NM newest-strike doesn't mask a closer one that's a few seconds older. Format: Closest strike: 1.9 NM @ 224° (SW) · 14s ago · 0 ≤5NM · 0 ≤10NM · 5 ≤25NM · 47 ≤50NM (last 10 min). The "X seconds ago" countdown ticks every second.

The yellow EXPERIMENTAL badge in the expanded view flags this whole feature (counts, motion detection, per-stagefield pips) as in active development. Treat as situational awareness, not authoritative.

Per-stagefield ⚡ pips

The banner is KOZR-centric. But the four stagefields (Allen ~4 NM SE, Toth ~8 NM ESE, Highbluff ~7 NM S, Lucas ~17 NM W) can have lightning in their 5 NM ring while KOZR's is clear — especially for Lucas, since it's far enough out west that storm cells in the western training area can hit Lucas without ever entering the KOZR ring.

Each Training Status panel (Cairns + 4 stagefields) gets its own small ⚡ pill in the top-left corner showing the closest strike's distance and compass cardinal direction (e.g. 2.1NM W or 7.8NM SE). Two tiers:

  • Danger — flash within 5 NM of the station. Pulsing red border. Operationally significant for that specific airfield.
  • Caution — flash within 10 NM (but not 5). Static yellow glow. Watch carefully; the next strike may be inside the 5 NM ring.

Pip is hidden when there are no flashes within 10 NM of that station. Hover for the full count + bearing in degrees. The publisher computes per-stagefield counts in a single pass through the same flash data — no extra fetches.

Storm motion / approach prediction

When the lightning service detects a coherent storm cell with enough flashes to track its motion, it adds an approach line below the detail line:

🌩️ Cell approaching at 28 kt — ~14 min to 5 NM ring MODERATE

The publisher clusters strikes by spatial proximity (within 5 NM = same cell), bins them into 5-minute time buckets, fits a least-squares motion vector to the centroid trajectory, and projects forward. ETA is computed analytically (quadratic intersection of the projected path with each radius ring). The line goes red and shows a 🚨 icon when the projected ETA to the 5 NM ring is ≤15 min, regardless of the active banner tier — so an inbound cell stands out even when there's no current strike inside 10 NM.

Confidence badges (HIGH / MODERATE / LOW) reflect flash count, bucket coverage, and trajectory linearity (R²). Motion analysis is conservative: requires ≥10 flashes in a cell across ≥3 time buckets and R² ≥ 0.5 before reporting any prediction. Cells with motion vectors pointing away from KOZR are skipped. The dashboard caps projections at 30 minutes — beyond that, atmospheric dynamics make the prediction unreliable.

Operational caveat: This is awareness, not prediction-as-fact. Real storm cells split, merge, intensify, and dissipate in ways pure linear motion can't capture. A wrong ETA is worse than no ETA — when in doubt, the system stays silent.

Data source: GOES-19 Geostationary Lightning Mapper via NOAA's public AWS Open Data bucket. Server-side fetch every 2 minutes (independent service, separate from the main 5-min publisher) — no per-viewer downloads or browser parsing. The dashboard re-polls lightning.json every 90 s, deliberately offset from the publisher's 120 s tick so the browser sees a fresh payload within ~30 s of each publish. End-to-end strike-to-banner latency typically 2–3 min, worst case ~5 min.

Active & Upcoming Advisories

WWA = Weather Watch, Warning, or Advisory. Pulled from the MEF Block 16.

  • AWA · Area Weather Advisory — within 60 NM of Cairns. Badge color AMBER.
  • TWA · Terminal Weather Advisory — within 15 NM of Cairns. Badge color RED.
  • WATCH — severe conditions possible. Amber.
  • WARNING — severe conditions imminent or occurring. Red.

The card only appears when at least one advisory is active (current Zulu time within its valid window) or upcoming (starts within the next 6 hours). Each active advisory shows an expiry countdown (e.g. Expires in 2h 15m). Upcoming advisories show a blue banner above the active list with a "Starts in Xh Xm" countdown.

Special time tokens CURR (currently active) and UFN (until further notice) are recognized — they're treated as open-ended and always active.

7-Day Outlook

The center of the VFR banner has a row of 7 day letters, each with a colored dot. Each dot = the worst of Ceiling / Visibility / Surface Wind across that day's AM/PM/N1 periods.

  • 🟢 GO — both ceiling and visibility are above the standard mins all day
  • 🟡 MARGINAL — at least one category drops into the marginal band (700 ft / 2 SM for CIG/VIS)
  • 🔴 NO GO — at least one category drops below minimums (500 ft / 1 SM or worse)

Click the widget to expand a detail panel below the banner showing:

  • The forecaster's synopsis — plain-English summary of the week
  • Full grid of 7 categories × days × periods (Ceiling, Visibility, Surface Wind, Turbulence, Icing, Thunderstorms, UAS Winds)
  • Dashed vertical lines separating each day
  • Legend at the bottom

The widget only drives its dot color from CIG/VIS/WIND; other categories are visible in the grid but don't factor into the banner rollup.

KOZR TAF Strip

Compact widget sitting in the VFR Operations banner next to the 7-Day Outlook. Shows the next 6 forecast hours from the live KOZR Terminal Aerodrome Forecast with a colored dot per hour.

Dot colors use FAA flight categories (not the 7-Day's GO/MARGINAL/NO-GO rollup) so pilots can read exact category at a glance:

  • VFR
  • MVFR
  • IFR
  • LIFR magenta dot

Click the widget to expand a detail panel directly below the banner with:

  • Next 12 hours (twice the banner view) as a horizontal row of hour + dot + category-label cells
  • Full raw TAF text in a monospaced block for reference
  • Issue time

Uses the aviationweather.gov TAF API's pre-decoded fcsts groups (no raw TAF regex on the client). For KOZR's military TAF, the publisher falls back to NWS tgftp text-product feed and parses BECMG / FROM / TEMPO / PROB groups locally. Widget hides entirely when KOZR has no published TAF (rare — KOZR issues 30-hour military TAFs every 6 hours at 00/06/12/18Z).

Worst-case per-hour color

The dot color at each hour is the worst of the active main group (BECMG / FM / initial) AND any active TEMPO / PROB overlay. Matches aviationweather.gov's display convention — so a TEMPO TSRA with 2 SM visibility shows up as IFR even when the main group has 6+ SM and only mid-level clouds. Operationally this means the strip surfaces hours where conditions COULD drop to IFR/LIFR, not just the predominant forecast.

In the expanded 12-hour view, hours where a TEMPO/PROB overlay is driving the worse color get an asterisk on the category label (e.g. IFR*) with a hover-tooltip explaining it. Severity ranking: VFR < MVFR < IFR < LIFR.

State persists via localStorage.tafOpen so the TV keeps the expanded view across the nightly 03:00 reload.

Training Status Panels

5 panels — Cairns AAF + 4 stagefields (Allen, Toth, Highbluff, Lucas). Each panel shows:

  • Top light bar glowing green/red/yellow based on GO/NO-GO/UNKNOWN
  • Station name in tracked uppercase
  • Big status text — VFR GO / NO GO / UNKNOWN
  • Subtitle line — current ceiling/vis for Cairns; area name + trainability countdown for stagefields
  • Recommended landing direction — favorable runway + headwind/crosswind
  • SVFR Recovery tier gauge (Cairns only, bottom row) — three segments showing which of the sliding-scale tiers (≥700/1 · ≥600/2 · ≥500/3) the live KOZR METAR currently satisfies. Each met tier lights green; unmet tiers stay dim gray. If zero tiers are met, the label turns red. Yellow when METAR missing or stale. Lives on Cairns because stagefield recoveries all happen into Cairns, and the per-tier display tells you how much buffer you have if ceiling or visibility degrade.
  • SOLO badge (stagefields only, AM/PM only) — teal SOLO GO or orange NO SOLO with reason
  • Advisory pip — small pulsing ⚠️ in the top right if an active advisory affects that stagefield's area; hover for the full advisory text

Cairns logic is pure VFR — green if ≥1000/3 from the live METAR, red below, yellow when METAR is missing or stale. IFR is handled by a separate strip in the IFR section.

Stagefield logic is area-forecast driven. Mapping: Allen → GOLDFISH, Toth → GOLDFISH, Highbluff → BEARCAT, Lucas → VANGUARD C. The panel reads the current active segment of the area forecast and checks ≥700/1.

Recommended Landing Direction

Each VFR training panel shows the runway most favorable right now based on the 1,000 ft MSL flight-level wind (the lowest altitude the MEF reports).

Format: RWY 24 ↓ HW 12kt ← 5kt

  • ↓ green — favorable headwind · ↑ red — tailwind (bad)
  • ← arrow — right crosswind (wind pushing aircraft left)
  • → arrow — left crosswind (wind pushing aircraft right)
  • ↔ — variable wind direction
  • Arrow turns orange when crosswind ≥ 15 kt
  • RWY CALM — wind speed zero
  • RWY VRB ↔ Xkt — variable wind direction

Runways hardcoded: Cairns 06/24 + 18/36, Allen 12/30, Toth 07/25, Highbluff 10/28, Lucas 18/36.

Solo Operations

Student solo weather rules are stricter than standard training minimums and use both observed and forecast weather. Solo badges appear on stagefield panels for AM and PM only (no solos at night).

  • Ceiling ≥ 1,200 ft AGL (BKN/OVC)
  • Visibility ≥ 3 SM
  • Surface wind ≤ 15 kt (including gusts)
  • Gust spread ≤ 10 kt (gust − sustained)
  • Any active or upcoming WWA mentioning wind > 15 kt whose valid window overlaps the training period = automatic NO SOLO
  • The rule is applied to the entire forecast period, not just the current minute. If any future main segment drops below limits, the panel shows NO SOLO with "(later)" in the reason.

Reason codes: Ceil 900ft, Vis 2SM, Wind 18kt, Gust G20, Spread 12kt, AWA Wind.

WAAS / GPS Coverage

Compact widget centered in the IFR Operations banner. Shows the live FAA Vertical Protection Level snapshot from nstb.tc.faa.gov with a colored pip indicating coverage at Cairns AAF, the local time of the last successful fetch, and a soft blue glow around the entire widget when coverage is at the LPV200 best-case state:

  • KOZR Full LPV200 — deep-blue zone, best WAAS service. LPV200 (200-ft DH) approaches available. Banner glows soft blue.
  • LPV — slightly elevated VPL but still within RNP service.
  • LNAV/VNAV — too high for full LPV but baro-VNAV approaches still flyable.
  • Marginal — verify alternate procedures.
  • Degraded — outside published service contour. Plan non-GPS approach.
  • WAAS unavailable — all CORS proxies failed. Hover for the last error string. Widget retries on the next 3-min cycle.

Click the widget to expand a detail panel showing the full FAA snapshot image, classification analysis, sampled RGB values, and the fetch timestamp.

Snapshot time shown in the banner is the local time of the last successful proxy fetch (24-hour HH:MML). FAA updates the source image every 3 minutes server-side; we re-fetch on the same cadence.

Persistent across refreshes — the last successful coverage classification is cached in localStorage, so reloading the page shows the previous status instantly while the fresh fetch runs in the background. Only the lightweight classification (category + RGB + timestamp) is cached, not the image itself.

How it works: the FAA renders a color-coded VPL contour map every 3 min. We pull it through one of three CORS proxies (allorigins.win → corsproxy.io → codetabs, in order — first one to return a valid PNG wins). The image is drawn onto an HTML5 canvas and a 17×17 pixel region is sampled around Cairns's coordinates (31.27°N, 85.72°W). White state-border pixels are dropped before averaging the RGB. The averaged color is classified into one of the categories above. Calibrated against NYC/LA/Miami/Seattle (all return deep blue) and Pacific/N. Atlantic control points (return deep red).

Alternate Planning

Thin sub-header under IFR Operations. Shows 5 commonly-used alternates evaluated against the active brief's DD 175-1 aerodrome forecast — not live METAR. Real alt-planning per FAR/USAACE filed-alternate rules requires forecast conditions, not current observations.

KMGMMontgomery RegionalILS RWY 28600 ft / 1¼ SM

How each alt finds its forecast (3-pass lookup):

  1. Direct match — airfield ICAO appears as its own aerodrome on the brief
  2. Sibling match — ICAO listed in another aerodrome's sameAs array (e.g. "Same as KOZR: KCEW, KDHN, KTOI, KMGM")
  3. Umbrella match — sameAs contains ALL STATIONS WITHIN 100NM (common on N1 briefs that list only KOZR with the umbrella flag — all 5 alternates are within 100 NM so they all ride it)

Worst-case evaluation: each cell uses the most restrictive ceiling AND most restrictive visibility across every period covering the current Zulu time — main / predominant + any FROM that's taken effect + any active TEMPO overlay. A TEMPO that drops below mins disqualifies the alt even if the predominant group is fine (matches aviationweather.gov's TAF display convention).

Forward timeline — below the current ceil/vis line each cell renders every upcoming worst-case segment in the active brief: countdown · projected ceil/vis (colored by FAA flight category) · POSS. ALT / POSS. ALT badge. So you see 3000/5 GO now → 2000/5 GO in 1h 30m → 500/1 NO GO in 2h 30m at a glance. Hidden when conditions are stable through brief expiry.

  • POSS. ALT (green) — worst-case ceiling AND visibility ≥ published alternate mins for that segment
  • POSS. ALT (red) — worst case falls below
  • UNKNOWN — no aerodrome forecast covers this airfield, or the brief doesn't cover the current Zulu time

Fallback text in the cell: No forecast (airfield not covered) / Outside brief window (no period covers now — e.g. between PM brief expiry and N1 start).

Old-weather overlay — when the active brief is operationally stale (issued previous day OR valid period expired), every alt row gets the same yellow diagonal-stripe ⚠️ OLD WEATHER overlay as the other forecast-driven sections.

Data Refresh & Auto-Update

  • Forecast + METAR data re-fetched every 5 minutes automatically. Live METAR is always compared against the cached one and only replaces it if newer.
  • Period auto-snap re-evaluates every 1 minute — AM→PM→N1 transitions happen without refresh.
  • Dashboard auto-reload — every 5 minutes the page checks GitHub Pages for a new index.html (via ETag). When a new version is detected it silently reloads so the TV picks up pushed changes without human intervention.
  • Daily reload — full page reload at 03:00 local to prevent memory bloat on always-on TV displays.

Old Weather Warning

If the currently displayed forecast was issued on a previous calendar day (operator's local time) — i.e., today's AM/PM/N1 brief hasn't been published yet and you're still looking at yesterday's — the dashboard enters an OLD WEATHER mode:

  • Yellow 3-pixel border around the entire viewport (same style as Demo mode)
  • Yellow banner up top reading ⚠️ OLD WEATHER — [PERIOD] brief from [date] · X days ago · Today's brief not yet issued

Primary signal is the mef.date stamped on the brief itself. Fallback is the fetchedAt / exportedAt timestamp, which catches the rare case where the iOS publisher itself stopped pushing. Demo mode overrides this — if you enable Demo, the demo yellow border takes priority and the old-weather banner hides.

Clears automatically the moment today's brief is parsed and pushed.

Mobile / Phone View

The dashboard is fully responsive. Two breakpoints kick in below the desktop layout:

  • ≤ 768px (tablet / phone landscape): Navbar wraps — brand left, action buttons right, datum chips (Forecast Date, Zulu, Local, Published heartbeat, ⚡ Detector) drop to their own row underneath. Section banners wrap so the 7-Day Outlook + TAF + WAAS widgets stack onto a second banner row at full width. Training panels switch to a 2-column grid; alternates also 2-column. METAR decoded cells wrap. Flight Level Winds becomes horizontally scrollable. Map iframes shrink to 260px tall.
  • ≤ 500px (phone portrait): All multi-column grids collapse to 1 column. Everything single-stack and readable on a phone screen.

Sticky navbar on mobile — once you scroll, the navbar pins itself to the top of the viewport so the brand, all timestamps, and the heartbeat pip stay visible. Background extends through the iPhone notch / status-bar safe area so nothing transparent shows behind the iOS clock.

The TV/desktop layout above ~1100px is completely unchanged — every mobile rule lives inside @media queries.

Pilot Reference Disclaimer

The muted gold strip at the very top of the page reads:

FOR PILOT REFERENCE ONLY — IT IS THE PILOT'S RESPONSIBILITY TO CHECK OFFICIAL SOURCES WHEN DETERMINING GO/NO-GO STATUS FOR TRAINING

Always visible regardless of demo / old-weather / amendment state. Reduced visual weight (low-saturation antique gold, smaller font, slim padding) so it acknowledges itself without dominating the live data below. Scales up on TV mode, down on mobile, stays one line where possible.

This dashboard scrapes and parses public weather products from army.mil/rucker + live METAR/TAF from aviationweather.gov + the FAA WAAS snapshot from nstb.tc.faa.gov. While every effort is made to render the data accurately, parsing errors, source outages, and processing delays are possible. Always cross-check against the original PDFs and live forecaster briefings before launch.
